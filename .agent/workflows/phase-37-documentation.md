# Phase 37 — Documentation Pass

**Owner:** Rahul
**Type:** Research + writing
**Depends on:** Phase 36

## Goal

Docs for users (setting up the keyboard, importing stickers, using
transfer/sharing) and for contributors (architecture overview, how to
build, how to add a theme/layout/dictionary) — "anyone can fork it" only
holds if they can actually understand it.

## Tasks

1. User-facing README/setup guide.
2. `CONTRIBUTING.md` covering how to propose a theme, layout, or
   dictionary addition.
3. An architecture doc summarizing the decisions made across all 37
   phases — a living document, not a one-time snapshot, since it should
   stay accurate as the project evolves post-launch.

## Definition of done

- [ ] Someone who has never seen this project can, from the docs alone,
      install the keyboard and know where to look to contribute a change.
